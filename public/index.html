Endpoints

All endpoints are prefixed with /geo/v1 and return JSON.

EndpointReturns
GET /geo/v1/states All 13 states and 3 federal territories, with capitals, regions, and ISO 3166-2 codes.
GET /geo/v1/states/{id|name} One state by ID (10) or name (selangor), including its districts.
GET /geo/v1/districts All 159 administrative districts nationwide.
GET /geo/v1/districts/{id|name} Districts of one state — e.g. /districts/sabah or /districts/12.
GET /geo/v1/mukims All 1,944 sub-districts. Filter by kind with ?type=mukim, ?type=bandar, or ?type=pekan.
GET /geo/v1/mukims/{id|name} Sub-districts of one district — e.g. /mukims/klang or /mukims/76. Same ?type= filter.
GET /geo/v1/search/{query} Case-insensitive search across states and districts. Filter with ?type=states, ?type=districts, or ?type=mukims.
GET /health Service status and uptime.

Use it anywhere

No key, no sign-up. Responses are cached and safe to call from the browser.

// Populate a state dropdown
const res = await fetch("/geo/v1/states");
const { data } = await res.json();

data.forEach(({ id, name }) => {
  stateSelect.add(new Option(name, id));
});

// Then load districts when a state is picked
const districts = await fetch(`/geo/v1/districts/${stateSelect.value}`)
  .then(r => r.json());
